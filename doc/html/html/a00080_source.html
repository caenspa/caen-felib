117 # Load variadic API
118 # Notes:
119 # - Remember to manually apply default argument promotions when calling variadic functions;
120 # anyway it is not necessary with ReadData since variadic arguments are always pointers,
121 # that are not subject to default argument promotion. More details on
122 # https://en.cppreference.com/w/c/language/conversion#Default_argument_promotions
123 # - On some platforms, like Apple ARM64, "it is required to specify the argtypes attribute for
124 # the regular, non-variadic, function arguments" (see ctypes documentation); the other
125 # variadic arguments could be set at runtime after a call to SetReadDataFormat, but this
126 # would not be safe because ReadData can be called concurrently from multiple threads with
127 # different signatures, so arguments (always pointers in this case) must be placed without
128 # relying on ctypes automatic conversions with from_param methods. For more details, see
